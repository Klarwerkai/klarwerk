# Modul: i18n — Internationalisierung

> Quelle: Pflichtenheft §3.13 (FR-I18N-01…02). Jira-Epic: KW-I18N.

## Ziel
Vollständige Zweisprachigkeit DE/EN inkl. KI-Antworten, mit Architektur für weitere Sprachen.

## User Stories & Akzeptanzkriterien

### FR-I18N-01 · DE/EN vollständig (MUSS)
- [ ] **Gegeben** der Umschalter, **dann** sind Oberfläche **und** Reasoner-Antworten/Interview in beiden Sprachen vollständig.

### FR-I18N-02 · Erweiterbar (SOLL)
- [ ] **Gegeben** eine neue Sprache, **dann** ohne Code-Umbau ergänzbar (Ressourcen-basiert).

## API / Schnittstellen (Entwurf)
Sprach-Ressourcen je Locale; Reasoner-Prompts sprachparametrisiert; `Accept-Language`/Nutzerpräferenz.

## Datenmodell
`users.locale`. UI-Strings in Ressourcendateien.

## Nicht-Ziele (v1)
Rechts-nach-links-Sprachen, automatische Maschinenübersetzung von KOs.

## Offene Fragen
Übersetzung bestehender KOs (Inhalt) vs. nur UI · Locale-Fallback-Regeln.
