KLARWERK Insel — Aufbau & Betrieb (KLLM-70 / KLLM-61)
=====================================================
Zwei Werkzeugsaetze in einem Paket:

A) AUFBAU/DOKU  (KLLM-70)  — Rechner reproduzierbar machen
   - Insel-inventarisieren.command : erfasst den Ist-Zustand des Mac und legt
     ihn versioniert im Repo ab (docs/operations/INSEL-AUFBAU.md +
     docs/operations/insel-inventar/). OHNE Secrets.
   - Insel-aufbauen.command : baut aus dem Inventar einen vergleichbaren
     Rechner (idempotent, Air-Gap-bewusst).

B) BETRIEB  (KLLM-61)  — die KLARWERK-App auf dem Mac starten
   - Insel-App-starten.command : faehrt die App NATIV hoch (kein Docker),
     bindet den lokalen LLM an und oeffnet den Browser. Der Server liefert
     die Web-Oberflaeche gleich mit aus (eine Adresse, http://127.0.0.1:3001).
   - Insel-App-Desktop-Icon.command : legt einmalig eine Doppelklick-App
     "KLARWERK App" auf den Schreibtisch, die den Launcher startet.

APP STARTEN — Schnellstart
   1) Terminal:
        bash ~/Documents/dev_Klarwerk/scripts/insel/Insel-App-starten.command
      -> Browser oeffnet http://127.0.0.1:3001
   2) In der App unter „KI-Verwaltung" den lokalen LLM als aktives Backend
      waehlen (der Launcher hat ihn bereits verbunden).
   3) Fenster offen lassen; Strg-C beendet die App.

   Bequemer: einmal Insel-App-Desktop-Icon.command ausfuehren -> danach
   startest du die App per Doppelklick vom Schreibtisch.

EINSTELLUNGEN (oben in Insel-App-starten.command)
   BACKEND=ollama|mlx        Standard: ollama (qwen3:32b)
   OLLAMA_MODELL=qwen3:32b   genaues Ollama-Modell
   MLX_MODELL=...            MLX-Modellname (Server auf :8080)
   PERSIST=journal|memory    journal = Daten bleiben; memory = frisch je Start
   PORT=3001                 Adresse

   Beispiel MLX statt Ollama, ohne Datei-Aenderung:
     BACKEND=mlx bash ~/Documents/dev_Klarwerk/scripts/insel/Insel-App-starten.command

VORAUSSETZUNGEN
   - Node.js (>=20) und die Projekt-Abhaengigkeiten (npm install) vorhanden.
   - Fuer Ollama: Modell geladen (ollama pull qwen3:32b). Fuer MLX: Server laeuft.
   - Das Frontend wird beim ersten Start automatisch gebaut, falls dist fehlt.

SICHERHEIT
   - Kein Cloud-Schluessel noetig; laeuft rein lokal (127.0.0.1).
   - Keine Secrets in Repo/Skripten/Doku. Schluessel bleiben im Schluesselbund.
   - Datenbank bewusst nicht in der Cloud: nativ, In-Memory oder lokales Journal.

ANDERER REPO-PFAD?
   export KLARWERK_REPO="/pfad/zum/klarwerk"   # vor dem Command setzen

Quelle der Wahrheit: KLLM-61 (App an lokalen LLM), KLLM-62 (Insel), KLLM-70 (Aufbau).
